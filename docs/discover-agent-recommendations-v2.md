## 0. Executive summary

**Что сейчас.** /discover — 20 адаптивных вопросов → Holland-код + Big Five + 3 сильные стороны + 8–12 профессий + план на 3–6 месяцев. Технически красиво, эмоционально ценно, коммерчески — тупик: пользователь закрывает вкладку и ничего не делает.

**Что меняем.** Каждый вопрос становится двойного назначения — кормит личностную модель **и** раскрывает жизненный контекст (есть ли клиенты, дети, пожилой родитель, хроника, публика, ресёрч-нагрузка, хаос в задачах). На выходе кроме профиля добавляется **главная секция: «Твоя связка агентов Finekot»** — 1–3 агента из каталога с честным матчем, списком задач, которые агент заберёт у пользователя, и roadmap-ом внедрения на 3 месяца.

**Чего не делаем.** Не подкручиваем результаты. Если сигнал слабый — так и говорим: «чёткой связки не видно, приходи в чат с Ada». Не пушим Pro-тарифы там, где достаточно Basic. Не рекомендуем больше 3 агентов одновременно.

**Масштаб изменений.** Два системных промта переписываются целиком. В Profile-схему добавляются два поля. В UI — одна новая hero-секция и переупорядочивание. Бэкенда и новых API не нужно. Stash-API подправляется, чтобы не ронять запрос при новых полях.

## 1. Почему именно так — проверка логики

### 1.1 Personality ≠ situation
Holland/Big Five рассказывают, **какой человек**, но не **что он делает**. Можно быть высоким по S (Social) без престарелых родителей — Eva не подойдёт. Можно быть E-предпринимателем без существующего бизнеса — David преждевременен. Значит нельзя мапить RIASEC→агент напрямую: это даст псевдо-рекомендации и подорвёт доверие (пушишь Eva человеку без родителей → всё, ушёл).

**Вывод.** Обязателен явный life-context слой. RIASEC/OCEAN используется как **усилитель** рекомендации (как агент впишется в характер), а не как её первоисточник.

### 1.2 Честность как продуктовое требование
Командир сформулировал: «мухлевать нигде не надо». Это не вежливая ремарка, это защита доверия к продукту. Переводим в инвариант:

- **Порог релевантности.** Агент попадает в рекомендации только если match ≥ 65. Иначе — нет.
- **Ноль-рекомендаций — валидный выход.** Если ни один не прошёл порог, секция показывает «недостаточно сигнала» + CTA в live-чат с Ada. Это честнее, чем вручную добрать.
- **Bugdet-discipline.** Default — 1 агент. 2 — если чётко разнесённые зоны жизни (например creator + small business). 3 — только когда три зоны бьют одновременно. 4+ — никогда.
- **Tier-discipline.** Basic по умолчанию. Pro — только если в ответах есть сигнал на агентство / команду / несколько детей / семью с общим vault.
- **Transparent math.** В карточке связки всегда видна сумма $/мес.
- **Evidence-first.** Каждая рекомендация приводит 1–2 конкретные цитаты из ответов пользователя (как сейчас делается в `strengths.evidence`).

### 1.3 20 вопросов хватит
Было: 7 Holland + 7 Big Five + 6 deep = 20.
Стало: 5 Holland + 4 Big Five + 2 anchor-situational + 9 adaptive-deep = 20.

Ключевой ход — **Q2 = anchor-вопрос** с multi-select по жизненным зонам. Он за один такт отсеивает 60% пространства ответов и даёт LLM-questioner направление, куда углубляться в Q3-Q20. Дальше LLM сам подбирает вопросы, которые одновременно закрывают нужную шкалу Big Five/RIASEC и углубляют уже увиденный контекст.

### 1.4 Overlap агентов — не делать двойных рекомендаций
- **Orban vs Boris.** Orban ($50 one-time) — голос → структурированные задачи. Boris ($49/мес) — это умеет **плюс** инбокс, календарь, morning brief, память. Правило: если Boris в связке — Orban не рекомендуется отдельно. Orban уместен, когда сигнал узкий: «хаос в задачах, ловлю голосом», без нагрузки на встречи и инбокс.
- **Eva vs Patrik.** Не пересекаются. Eva — уход за отдельно живущим родителем. Patrik — личное здоровье + mental-health. Могут сосуществовать (в Patrik Pro есть «optional sync with Eva»).
- **Ada vs Hanna.** Ada исследует, Hanna пишет. Частая связка для креатора-аналитика.
- **David vs Boris.** Boris обслуживает **тебя как человека**, David — **твой бизнес**. Solo-фаундер без сотрудников получает первый David, без Boris. Исполнительный директор компании, нанятый — Boris, без David.

### 1.5 Roadmap > bundle
Пушить «бери сразу 3 агента на $200/мес» = закрыть воронку. Правильно: «начни с одного сейчас, через 30 дней добавь второго, через 90 — оценим третий». Это снижает порог входа и даёт нам сценарий возврата пользователя (trigger-mail через месяц: «пора добавить Hanna?»).

### 1.6 Кто главный потребитель результата
Результат /discover читают **двое**: человек и ADA-агент в чате на сайте (через persist / TG-стэш). Поэтому профиль должен быть одновременно:
- **читаемым человеком** — карточки, проценты, иконки, human summary;
- **структурированным для Ada** — JSON с `agentRecommendations`, которым ADA может оперировать в диалоге: «Я вижу, у тебя сигнал на David 82% — расскажу подробнее?»

Это делается одним и тем же JSON, рендер — два разных.

## 3. Редизайн вопросов

### 3.1 Структура бюджета

```
Q1   — opener RIASEC (single, 6 options) — грубое позиционирование
Q2   — ANCHOR life-context (multi, 8 options) — ключевой отсев, задаёт Q3-20
Q3-5 — Holland-добор + life-context detail
Q6-9 — Big Five scales (scale 1-5)
Q10  — OPEN text — «что самое главное, с чем ты сейчас буксуешь?» (свободный ввод, ≤ 3 за весь квиз)
Q11-17 — adaptive deep по зонам, выбранным в Q2 + раскрытым в Q10
Q18-19 — budget/decision context (один single + один scale)
Q20  — closing text — «опиши идеальный следующий квартал одним абзацем» (свободный ввод)
```

### 3.2 Примеры ключевых вопросов

**Q1 (single, RIASEC-opener):**
«Представь идеальный рабочий день. Что ты делаешь большую часть времени?»
- Руками делаю что-то физическое (R)
- Разбираюсь в сложной задаче / данных (I)
- Создаю что-то новое — текст, визуал, идею (A)
- Работаю с людьми — помогаю, учу, договариваюсь (S)
- Двигаю проект / продаю / веду команду (E)
- Навожу порядок в системе — цифры, процессы, документы (C)

**Q2 (multi, ANCHOR — самый важный вопрос):**
«Отметь всё, что сейчас реально ест твоё время или держит в голове:»
- клиенты / собственный бизнес
- дети 5–15 (своё/неё, ваше)
- пожилой родитель, живёт отдельно
- моё здоровье — хроника, тревожность, анализы
- встречи и входящие сообщения (инбокс-хаос)
- публикации в соцсетях / строю личный бренд
- ресёрч, анализ, конкуренты, лит-обзор
- задачи расползаются, ловлю голосом
- ничего из перечисленного

**Q10 (text, mid-quiz):**
«Одно предложение: с чем ты сейчас больше всего буксуешь? Что сжирает энергию, но не двигается?»

**Q18 (single, budget context — честный):**
«Если агент заберёт у тебя 10 часов в неделю, за какую сумму в месяц это честная сделка?»
- до $30
- $30–80
- $80–150
- $150–300
- $300+ / не вопрос цены

**Q19 (scale):**
«Я сейчас готов начать использовать нового AI-помощника уже на этой неделе»
(1 = думаю полгода наперёд, 5 = завтра поставлю)

**Q20 (text, closing):**
«Опиши одним абзацем: какой должен быть твой следующий квартал, чтобы ты был им доволен?»

### 3.3 Правила генерации Q3–Q17 (адаптивные)

Передаются в QUESTIONER_SYSTEM_PROMPT:
- После Q2 LLM должен знать, какие зоны помечены. Если пользователь выбрал «дети 5–15» — минимум 1 вопрос про стиль воспитания/язык/homeschooling. Если выбрал «пожилой родитель» — минимум 1 про дистанцию / частоту видимости / здоровье родителя.
- Нельзя задавать вопросы о зонах, **не отмеченных** в Q2 (кроме случая, если предыдущий ответ явно намекает на скрытую зону — тогда 1 «проверочный» вопрос).
- Big Five-шкалы покрываются целиком — но через вопросы, встроенные в контекст зон (пример: «Когда клиент задаёт тупой вопрос в третий раз, что ты обычно делаешь?» → параллельно A и E-scale).

## 9. Edge cases / честность — чек-лист

4. ✅ Пройти квиз как «хочу попробовать AI, но боюсь подписок» (readiness = 2, budget до $30) → рекомендация Orban (one-time) либо fallback.

6. ✅ В ответах ни слова про детей → Taras.match < 65 гарантированно (LLM следует rubric).
7. ✅ Противоречивые ответы (юзер троллит) → fallback с честным reason.
8. ✅ Match 100 не возникает (потолок 95 для лучшего случая — оставляем место для сомнения).
